# Инструкция по работе с командами GIT

## Установка и первоначальная настройка

Первоначальная настройка
После установки необходимо настроить имя пользователя и email:
git config --global user.name "Ваше Имя"
git config --global user.email "ваш@email.com" 

Эти настройки будут использоваться для подписи всех ваших коммитов.
Флаг `--global` применяет настройки ко всем репозиториям на вашем
компьютере.
Для проверки текущих настроек используйте:
git config --list 

## Создание репозитория и основные команды

Инициализация репозитория
Существует два способа начать работу с Git:
1. Создать новый репозиторий:
git init
2. Клонировать существующий:
git clone https://github.com/username/repository.git

## Основные команды для работы с файлами
Проверка статуса репозитория:
git status
Эта команда показывает текущее состояние рабочей директории и области
подготовки.
Добавление файлов в индекс:
git add filename.txt # добавить конкретный файл
git add . # добавить все файлы
git add *.js # добавить все JS файлы
Создание коммита:
git commit -m "Описание изменений" 

## Просмотр истории

Просмотр логов:
git log # полный лог
git log --oneline # краткий формат
git log --graph # граф коммитов
Просмотр конкретного коммита:
git show [commit-hash]
Просмотр изменений:
git diff # изменения в рабочей директории
git diff --staged # изменения в индексе

## Работа с ветками

Концепция ветвления
Ветки в Git — это указатели на определённые коммиты в истории проекта.
Ветвление позволяет разрабатывать функции параллельно, не мешая
основной разработке. По умолчанию Git создаёт ветку `main` (или `master` в
старых версиях).
Основные команды для работы с ветками
Просмотр веток:
git branch # локальные ветки
git branch -a # все ветки (включая удалённые)
Создание ветки:
git branch feature-name 

Переключение между ветками:
git checkout feature-name
git switch feature-name # новая команда (Git 2.23+)
Создание ветки и переключение:
git checkout -b feature-name
git switch -c feature-name
Слияние веток:
git checkout main
git merge feature-name
Удаление ветки:
git branch -d feature-name # безопасное удаление
git branch -D feature-name # принудительное удаление

### Работа с удалёнными репозиториями

Добавление удалённого репозитория
git remote add origin https://github.com/username/repo.git
Здесь `origin` — стандартное имя для основного удалённого репозитория.
Основные операции с удалёнными репозиториями
Просмотр удалённых репозиториев:
git remote -v
Отправка изменений:
git push origin main
git push -u origin feature-name # с установкой upstream 

###Получение изменений:
git fetch origin # получить данные без слияния
git pull origin main # получить и слить изменения
Разница между Fork и Clone
Clone создаёт локальную копию репозитория на вашем компьютере. Если у вас есть
права доступа, вы можете напрямую отправлять изменения в оригинальный
репозиторий.
Fork создаёт независимую копию репозитория под вашим аккаунтом на GitHub/
GitLab. Это используется для:
• Вклада в открытые проекты
• Экспериментов без влияния на оригинал
• Создания собственной версии проекта
